# Звід правил front-end + архітектури (для будь-якого проєкту)

Зібрані принципи для нових проєктів. Початково сформульовані на OpenCart-зразку,
застосовні до будь-якого стеку (WordPress, Laravel, чистий PHP, SPA).

---

## A. Доставка ассетів і продуктивність (найважливіше)

1. **Critical CSS першого екрана — inline у `<head>`** через `<style>…</style>` (не `<link>`). Жодного render-blocking запиту для першого екрана.
2. **Решта стилів — окремо й відкладено**: `page.css` (critical) + `page-delayed.css` (решта) через `<link>` non-render-blocking (`media="print" onload="this.media='all'"` + `<noscript>`-фолбек).
3. **CSS бандлиться по типах сторінок**, а не один файл на весь сайт. Маршрут → свій бандл (home, product, category, information…). Користувач головної не вантажить стилі сторінки товару.
4. **Усі скрипти — `defer` і в кінці `<body>`.** Нічого не блокує парсинг.
5. **Сторонні віджети** (аналітика, чати, карти) — тільки `async` + динамічний `createElement('script')`. Ніколи не в критичному шляху.
6. **Cache busting через `?v=<version>`** на КОЖНОМУ css/js. Версія з конфіга (або `filemtime`) — одна точка інвалідації; кеш можна ставити агресивно (1 рік).
7. **У прод — `.min`-версії**, оригінал тримати поруч (`minified/` або окремо) для розробки.
8. **Персоналізований/динамічний контент** (кошик, вішліст, банери, лічильники) **довантажувати AJAX-ом після рендеру** — тоді статичний HTML можна агресивно кешувати.
9. **`loading="lazy"`** на всіх некритичних `<img>`; критичним (hero) — `loading="eager"` + `fetchpriority="high"` + явні `width/height` (проти CLS).
10. **`<link rel="preconnect">`** для зовнішніх доменів (шрифти, CDN).

## B. CSS / SCSS

11. **Шарова структура файлів** (7-1-подібна):
    - `general/` — `_variable`, `_helper` (міксини), `_general` (база/reset);
    - `block/` — переюзабельні компоненти (1 файл = 1 блок);
    - `area/` — секції сторінок (header, footer, home-page…);
    - `libs/` — сторонній CSS; `old-styles/` — легасі ізольовано окремо.
12. **Page-бандл = лише набір `@import` потрібних шарів.** Жодних стилів прямо в бандлі.
13. **Нуль «магічних» значень** — усе через токени у `_variable`: кольори (`$accent-color-1`, `$text-color`), відступи, межі (`$border-width`), шрифти, ширини (`$max-content-width`). Новий колір/відступ → спершу змінна.
14. **Строгий BEM**: `block__element`, `block--modifier` (`row-2__block-cart`, `icon-counter--filled`).
15. **Стан UI — класами на `body`/корені**, не inline-стилями: `header--sticky`, `show-menu`, `show-search`. JS лише тоглить клас, вигляд — у CSS.
16. **Повторювані патерни — у міксини** (`_helper`): `disable-link`, `hide-button`, `hide-list`, `assign-tooltip`. Не дублюй `list-style:none;margin:0;padding:0`.
17. **Контейнерна система через утиліти**: `.max-website-width` / `.max-content-width` / `.side-padding` — єдині обгортки ширини й відступів на весь сайт.
18. **`box-sizing: border-box`** на контейнерах; явний reset (`fieldset{border:0;padding:0;margin:0}`).
19. **`rem`** для розмірів макета (ширини, відступи); `px` лише де доречно (тонкі межі, дрібний текст).
20. **`@media (hover: hover)`** для hover-ефектів — щоб не залипали на тач-пристроях.
21. **Коментуй наміри оптимізації** (напр. які насиченості шрифту потрібні на першому екрані).
22. **Коміт компільованого CSS + `.css.map`** поруч зі `.scss` (бо сервер роздає готовий CSS).

> Адаптація під Tailwind: токени → `tailwind.config` (theme.extend); шари block/area → `@layer components`; BEM не застосовний (утиліти), але семантичні компоненти виносити в `@layer`; container-утиліта = одна `.container-site`.

## C. JavaScript

23. **Власний код — vanilla JS.** jQuery лише для легасі-бібліотек; новий код від нього незалежний.
24. **Один модуль = один файл = один об'єкт-неймспейс**: `var cart = { add(){…}, render(){…} }`. Глобальний скоуп чистий, методи згруповані.
25. **Перед викликом зовнішніх API — перевіряй існування**: `if (typeof gtag === 'function') gtag(...)`. Блокувальник не ламає сайт.
26. **Захищений доступ до DOM**: optional chaining `popup?.querySelector(...)` і `response.price ?? null`.
27. **Захист від подвійного сабміту** прапорцем: `if (flag) return; flag = true;` (знімається у відповіді).
28. **Стан — через `classList.toggle/add/remove`**, не через прямі стилі.
29. **Помилки серверу** — людяне повідомлення користувачу + `console.dir` для дебагу, а не «голий» `alert(status)`.
30. **Перевикористовувані шматки HTML — як шаблон-рядок** (`tpl.replace('{{x}}', …)`), не конкатенація по коду.
31. **Параметри з дефолтами** замість перевантажень: `add(id, title, price, image, qty = 1, type = '')`.
32. **Крос-модульні неймспейси для спільної логіки** (`PriceUpdater.updateProductPrices(...)`) — не дублюй розрахунки.

---

## D. HTML / розмітка / доступність

33. **`aria-label` на всіх іконкових посиланнях/кнопках** (`aria-label="cart"`); декоративні svg — `aria-hidden="true"`.
34. **Іконки — інлайн `<svg>`** (менше запитів, стилюється з CSS); декоративні `<img>` — з `alt` і `loading="lazy"`.
35. **`hreflang alternate`** у `<head>` для мовних версій.
36. **Schema.org / OGP генеруються на бекенді** і вставляються в `<head>` — SEO за замовчуванням.
37. **Семантика**: `<header>`, `<main>`, секції; sticky-хедер окремим блоком зі своїм контентом для скрол-стану.
38. **Дропдауни/меню — доступні з клавіатури**: відкриття кліком/фокусом, закриття Escape і кліком поза межами (не лише `:hover`).

---

## E. Зображення та шрифти

39. **WebP за замовчуванням**: усі ресайзи віддають WebP, оригінальний формат — лише за явним прапорцем.
40. **Шрифти самохост** (woff2, variable) + `@font-face` з `font-display: swap`; критичний файл — `preload`; курсив/некритичні насиченості — на вимогу.

---

## F. Кешування

41. **Повносторінковий кеш** готового HTML (TTL напр. 4 год) зі **списком винятків** (`skip_urls`: cart/checkout/account/особисті сторінки), опційно cache-by-device. У парі з правилом A8 (AJAX-довантаження динаміки) — основа швидкості.
42. **Redis/об'єктний кеш** як інфраструктура (predis тощо) — для запитів і об'єктів.

---

## G. Бекенд-сервіси (чисті межі)

43. **Централізована «грошова» логіка** в одному сервісі/класі (ціни, націнки, знижки, доставка) — `Price::calculateIncome(...)`, а не розмазано по контролерах.
44. **Шар типізованих повідомлень**: фабрика + handler (`error/info/success/warning`). Один сервіс нотифікацій, а не `echo`/`alert` по коду.
45. **Черга фонових задач** (payloads, delay, logger). Важке (парсинг, імпорт, листи) — у чергу, не в request.
46. **Кожна зовнішня інтеграція — окремий клас-обгортка** (Allegro, Nova Poshta, LiqPay, WayForPay, Translate…). Жодного `curl` у контролері: один API = одна бібліотека з чітким інтерфейсом.
47. **Централізоване логування** (єдина точка) + логер усередині черги. Не `error_log` врозсип. **Не писати PII у веб-доступні логи.**

---

## H. SEO як частина платформи

48. Не «прикручене», а вбудоване: **ЧПУ-роути, sitemap, canonical, hreflang, Schema.org/OGP з бекенду.** Закладай SEO-шар одразу, а не потім.

---

## I. Жива документація бізнес-логіки

49. Окрема доменно-структурована документація (`pages/` + `business-rules/`), що пояснює **поведінку, а не код**:
    - `pages/` — огляд кожної сторінки (маршрут → контролер/шаблон → блоки зверху вниз);
    - `business-rules/` — деталі по доменах (product, cart, checkout, search…).
50. Сильні риси доків (копіювати як стандарт):
    - посилання на реальні `file:line` (документ веде прямо в код);
    - перелінкування між доками (`→ business-rules/product/price.md`);
    - прив'язка до тикетів і дат (`з січня 2026 — TICKET-1199`);
    - сухий фактичний тон (описує що Є, без «як переписати»);
    - чесно фіксує пастки («дублюється у двох місцях», «умови різні — результат може відрізнятися»).

---

## J. Захисне програмування / помилки / типи

51. **Нормалізуй вхідні типи на межі функції**: `in_array($id, (array)$ids, true)` — приведення перед перевіркою, щоб не впасти на скаляр.
52. **Усвідомлена робота з помилками legacy**: глушити не наосліп, а точково й з поясненням під конкретну несумісність (`.user.ini` / `startup.php`: `E_ALL & ~E_DEPRECATED & ~E_NOTICE`, з коментарем чому).
53. **Не послаблюй типи без потреби**, але в колбеках фреймворку (де аргументи можуть бути не того типу) — обережно з суворими сигнатурами.

---

## K. Залежності / Composer / інфра

54. **Локальні path-пакети** для власних модулів (`type: path`) — монорепо-стиль без публікації; чисті межі модулів.
55. **Контроль залежностей** — секція `audit`; не тягнути зайвого.

---

## L. Дисципліна комітів і процес

56. **Кожна зміна — `TICKET-NNNN | English desc`**, і ці ж номери цитуються в доках.
57. **Код ↔ коміт ↔ документація ↔ тикет** утворюють зв'язний ланцюг — сильно для аудиту історії.
