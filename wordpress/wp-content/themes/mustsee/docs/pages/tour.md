# Сторінка: Тур (single product)

- **Маршрут:** `/product/<slug>/` (permalink товару WooCommerce)
- **Шаблон:** `woocommerce.php` (врапер) → `woocommerce/content-single-product.php` (наш макет)
- **Дані:** сам товар + мета `_mustsee_*` (див. [products.md](../business-rules/products.md)).

## Чому окремий шаблон

`woocommerce.php` рендерить single product **без** контейнера (full-width), решта Woo-сторінок — у `.container-site`. Перемикач: `is_product()` у `woocommerce.php`.

## Блоки

1. Хлібні крихти: Головна / Каталог / назва.
2. Галерея: головне зображення (`mustsee_media`) + до 4 мініатюр із галереї товару.
3. Мета-рядки: Країни / Виїзд / Місце / Тривалість / Місто / Зірки (тільки заповнені).
4. Ціна: `get_price_html()`.
5. CTA: **«Замовити тур» → `/booking/?tour=<id>`** (`$booking_url`, `content-single-product.php:21`) + «Версія PDF» (заглушка).
6. Найближчі дати виїзду: `mustsee_parse_departures()`.
7. Вкладки (`data-tabs`, `content-single-product.php:129`): Опис (контент товару) / Програма (`_mustsee_program`) / Калькулятор / Вартість для груп / Акції / Маршрут (`_mustsee_route`) / Готелі (товари кат. `hotels`) / Галерея.
8. Lead-форма + розсилка.

## Пастки

- Вкладки Опис/Програма/Маршрут/Галерея — **дані-залежні**: порожні показують текст-заглушку «уточнюється», вкладка не зникає.
- «Калькулятор цін», «Вартість для груп», «Акції» — наразі **статичні** (демо); ціна в калькуляторі береться з товару, решта полів — фіксовані.
- Вкладка «Готелі» показує товари категорії `hotels` як повʼязані (не привʼязані до конкретного туру).
- CTA веде в [бронювання](booking.md), а не одразу в кошик — додавання в кошик відбувається наприкінці флоу бронювання.
