# Бізнес-правила: Бронювання → кошик → каса

## Ланцюг

1. **Сторінка туру** → «Замовити тур» → `/booking/?tour=<id>` (`woocommerce/content-single-product.php:21`, `$booking_url`).
2. **Бронювання** (`page-templates/template-booking.php`): 3 кроки, вибір місць.
3. **Завершення** (`MustSee.booking`, `assets/js/theme.js:74`): додати в кошик → каса.

## Гроші

- Ціна туру = ціна товару WooCommerce (`get_price_html()`), валюта UAH.
- Комісія агента = `mustsee_commission_rate()` `functions.php:218` — **єдине джерело** (дефолт `0.12`, фільтр `mustsee_commission_rate`). Не хардкодити відсоток у шаблонах.
- Кількість у кошику = к-сть вибраних місць (мінімум 1).

## Додавання в кошик

Стандартний механізм WooCommerce, без кастомного бекенду:
```
{cart_url}?add-to-cart={product_id}&quantity={seats}
```
`assets/js/theme.js:161`. `cart_url` приходить з `data-cart-url` (`wc_get_cart_url()`), `product_id` — з `data-product` (`template-booking.php:37`). Кнопка захищена прапорцем від подвійного сабміту.

## Пастки / межі (станом на 2026-06-20)

- Вибір рейсу / типу розміщення / додаткових послуг — **демо-дані в шаблоні**, у кошик/замовлення поки **не передаються** (тільки сам тур × кількість). Це наступний крок.
- Дані туристів (крок 3) — форма без сабміту на сервер (демо).
- Кошик/каса WooCommerce 9 — **блоки** (рендеряться на клієнті), тож вміст кошика не видно через `curl` без сесії; механізм додавання — стандартний.
- Платіжний шлюз — `monobank-gateway` (плагін), тема в нього не втручається.
