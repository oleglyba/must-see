# Бізнес-правила: Продуктивність ассетів

## CSS: critical inline + решта deferred

- **Critical** (перший екран) збирається окремим конфігом `tailwind.critical.config.js` (сканує header/hero/search/картки) → `assets/css/critical.css`, **інлайниться в `<head>`** через `mustsee_inline_critical_css()` `functions.php:70`.
- **Повний** бандл `assets/css/tailwind.css` + Google Fonts + `style.css` — **non-render-blocking**: `mustsee_defer_styles()` `functions.php:88` (`media='print' onload="this.media='all'"` + `<noscript>`).
- `preconnect` до font-CDN — `mustsee_resource_hints()`.

## JS

- Усе через `wp_enqueue_script(..., strategy=defer, in_footer)` (`mustsee_assets()` `functions.php:43`).
- Власний код — vanilla, неймспейс `window.MustSee` (`assets/js/theme.js`): `menu`, `tabs`, `toggles`, `booking`. Без jQuery.

## Зображення

- **WebP за замовчуванням**: фільтр `image_editor_output_format` (`functions.php:199`) мапить jpeg/png → webp для всіх згенерованих розмірів (оригінал зберігає формат). Якість webp — 82 (`wp_editor_set_quality`).
- Hero — `loading=eager` + `fetchpriority=high` (LCP); решта — `loading=lazy` + `decoding=async` (`mustsee_media()` `inc/products.php:13`, `template-parts/hero.php`).
- Конвертувати наявні: `wp media regenerate --yes`.

## Full-page cache (WP Super Cache)

Повносторінковий кеш — плагін **WP Super Cache** (Simple-режим, `mod_rewrite=0`, без .htaccess; конфіг `wp-content/wp-cache-config.php`):

- TTL `cache_max_time = 14400` (4 год); `wp_cache_not_logged_in = 1` (залогінені — без кешу); `wp_cache_no_cache_for_get = 1` (будь-який `?query` не кешується → `?cat/?tour/?add-to-cart` завжди свіжі).
- `cache_rejected_uri` (винятки): `cart`, `checkout`, `my-account`, `booking`, `cabinet`, `login`, `register`, `add-to-cart` + `wp-*.php`.
- Інвалідація — автоматична (плагін чистить на збереження посту/товару/коментаря).
- WooCommerce-безпечно: кошик/каса/акаунт виключені, для залогінених і `?query` кеш обходиться.
- Перевірка: у HTML кешованих сторінок є коментар `<!-- Cached page generated by WP-Super-Cache ... -->`; статичні файли — `wp-content/cache/supercache/<host>/`.
- Ручне очищення: кнопка «Delete Cache» в адмінці (Settings → WP Super Cache) або видалити теку `wp-content/cache/supercache/`. Після зміни шаблонів/CSS варто очистити вручну.

## Інше

- Cache-busting: `?ver=filemtime` на CSS/JS.
- Hover лише на пристроях з hover: `future.hoverOnlyWhenSupported` (`tailwind.config.js`).
- Єдина контейнерна утиліта `.container-site` (ширина + бічні відступи) — у `assets/css/tailwind.src.css`.

## Збірка

```bash
cd wp-content/themes/mustsee
npm install            # node_modules не комітиться; відновлюється звідси
npm run build          # = build:critical + build:main
```
> ⚠️ Завжди запускай `npm run build` після зміни Tailwind-класів у PHP — інакше нові класи не потраплять у CSS. `node_modules/` видалено з диска для чистоти; перед збіркою — `npm install`.

## Свідомо НЕ зроблено (з причинами)

- Per-page CSS-бандли — purged-бандл уже ~23 KB, виграш мінімальний.
- Окремий `.min` для JS — файл малий.
- AJAX-довантаження персоналізації (кошик-лічильник у хедері) — у хедері лічильника поки немає; додати, якщо знадобиться показувати кошик на кешованих сторінках.
- Redis object-cache — інфра рівня сервера, не код теми.
